# Getting Started with Skills

**Welcome!** This guide helps you understand and use the skills system effectively.

---

## What Are Skills?

Skills are specialized tools for specific tasks in marketing, SEO, CRO, technical development, and more. Each skill:

- **Has clear documentation** with frameworks and examples
- **Solves a specific problem** or answers a specific question
- **Works independently** but connects with related skills
- **Includes examples** you can adapt for your use case
- **Follows a standard structure** so they're easy to learn

## Using Skill Workflows

Workflows guide you through multiple skills in a sequence for common scenarios.

### Step 1: Choose Your Workflow
1. Find a workflow that matches your goal
2. Review timeline and complexity
3. Assess resource availability

### Step 2: Plan Phases
1. Break project into phases (usually 3-5)
2. Assign ownership if team project
3. Schedule timeline

### Step 3: Execute Phase by Phase
1. Complete each skill in order
2. Validate outputs before moving to next phase
3. Document decisions and learnings

### Step 4: Track Progress
1. Measure success metrics at end
2. Compare to workflow expectations
3. Note what worked well or could improve

### Step 5: Iterate
1. Some skills may need multiple rounds
2. Be ready to adjust based on results
3. Monitor metrics post-completion

## Tips for Success

### 1. Start with Accessible Skills
- Copywriting and keyword research are good starting points
- Workflows are self-contained and don't require prerequisite skills
- Quick skills (10-30 min) are good for first attempts

### 2. Read Examples First
- Every skill has examples
- See how others have used it before attempting yourself
- Adapt examples rather than starting from scratch

### 3. Use Templates
- Most skills provide templates
- Copy them to save time
- Customize for your specific situation

### 4. Follow the Workflows
- Don't skip around randomly
- Workflows have sequential dependencies
- Each phase builds on previous ones

### 5. Validate as You Go
- Use validation checklists
- Check outputs before moving forward
- Fix issues immediately, not later

### 6. Reference the Docs
- Each skill has documentation
- Use references/ directory for deep dives
- Don't rely solely on memory

### 7. Monitor Results
- Track metrics defined in each skill
- Compare to success criteria
- Adjust approach based on data

## Common Workflows by User Type

### If You're Starting Fresh (New Business/Site)
1. **keyword-research** — Find opportunities (0.5-1 day)
2. **competitor-analysis** — Understand market (0.5-1 day)
3. **content-strategy** — Plan direction (0.5 day)
4. **seo-plan** — 6-month roadmap (0.5 day)
5. **seo-content-writer** — Create content (ongoing, 2-4 hours/page)
6. **copywriting** — Polish messaging (1-2 hours/page if needed)
7. **seo-audit** — Final validation (1 day)
8. **alert-manager** — Set up monitoring (1 hour)

**Total: 1-2 weeks for small site, 3-4 weeks for larger site**

### If You Have Existing Content (Improvement Mode)
1. **seo-audit** — Find issues (1 day)
2. **serp-analysis** — Understand rankings (0.5 day)
3. **content-refresher** — Update content (1-2 hours/page)
4. **copywriting** — Improve messaging (optional, 1-2 hours/page)
5. **technical-seo-checker** — Fix technical issues (0.5-1 day)
6. **alert-manager** — Update monitoring (30 min)

**Total: 1-2 weeks depending on page count**

### If You Want to Scale (Build Many Pages)
1. **programmatic-seo** — Plan strategy (1-2 days)
2. **seo-content-writer** — Create templates (1-2 days)
3. **schema-markup-generator** — Structure data (0.5 day)
4. (Development team builds pages from templates)
5. **seo-audit** — Validate pages (1-2 days)
6. **alert-manager** — Monitor pages (1 hour)

**Total: 2-4 weeks depending on page count**

### If You Want to Improve Conversions
1. **page-cro** — Analyze page (0.5 day)
2. **copywriting** — Improve copy (1-2 days)
3. **ab-test-setup** — Plan test (0.5 day)
4. (Dev team implements test)
5. **analytics-tracking** — Set up tracking (0.5 day)
6. (Wait for results, 2-4 weeks)

**Total: 1 week setup + 2-4 weeks testing**

## Key Principles

1. **Skills are self-contained** — You can use any skill without prerequisites (though some are easier with background knowledge)

2. **Workflows guide sequences** — Use workflows for complex projects that require multiple skills in order

3. **Examples inform usage** — Every skill has examples; adapt them for your situation

4. **Validation ensures quality** — Each skill has success metrics; use them to validate work

5. **Related skills provide alternatives** — When stuck, related skills section shows options

6. **Documentation is complete** — Every skill has what you need to succeed

7. **Reference materials go deep** — For complex topics, reference files provide detailed guidance

8. **Progress builds on phases** — Complete each phase before moving to the next
