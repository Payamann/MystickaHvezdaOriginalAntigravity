## 📝 Using Individual Skills

### Anatomy of a Skill

Every skill has:

1. **YAML Frontmatter** — Metadata (name, description, version, author, tags, triggers)
2. **Quick Summary** — What the skill does (1-2 sentences)
3. **When to Use** — Clear use cases and scenarios
4. **What It Does** — Main functions (3-5 points)
5. **How to Use** — Examples and step-by-step guides
6. **Core Concepts** — Key concepts with examples
7. **Frameworks** — Methodologies or structured approaches
8. **Examples** — 2-3 real-world scenarios
9. **Tips for Success** — 5+ practical tips
10. **Success Metrics** — How to validate it worked
11. **Related Skills** — Cross-references to other skills

### Working with a Skill

1. **Read** "When to Use" to confirm it's the right skill
2. **Review** "How to Use" examples for your situation
3. **Follow** step-by-step instructions
4. **Reference** frameworks and checklists as needed
5. **Check** "Success Metrics" to validate your work

## 💡 Best Practices

### 1. Start Simple
- Begin with quick skills (10-30 min)
- Master single skills before combining
- Progress to comprehensive skills as you gain experience

### 2. Follow Examples
- Every skill has examples
- Adapt examples for your situation
- Don't start from scratch

### 3. Use Workflows for Complex Projects
- Workflows have dependencies
- Complete phases in order
- Don't skip validation checkpoints

### 4. Reference Materials Are Your Friend
- Read reference materials for deep dives
- Use templates to save time
- Copy checklists for consistency

### 5. Validate as You Go
- Use success metrics defined in each skill
- Run validation checkpoints before moving forward
- Fix issues immediately, not later

### 6. Monitor Results
- Track metrics defined in each skill
- Compare to success criteria
- Adjust approach based on data

### 7. Leverage Related Skills
- Check "Related Skills" section in each skill
- These skills work well together
- Follow the patterns they suggest
